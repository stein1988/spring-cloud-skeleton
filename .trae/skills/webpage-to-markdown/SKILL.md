# 网页转 Markdown 文档

将网页内容转换为结构化的 Markdown 文档并保存到指定目录。

## 触发条件

当用户要求将某个网页内容保存为本地 markdown 文件时使用。

### 触发关键词（中英文）

**中文触发词**：
- 保存（网页/链接/文章/内容/网址）
- 下载（网页/链接/文章/内容/网址）
- 抓取（网页/链接/文章/内容/网址）
- 收藏（网页/链接/文章/内容/网址）
- 导出（网页/链接/文章/内容/网址）为 markdown
- 把（网页/链接/文章/内容/网址）保存为 markdown

**英文触发词**：
- save (webpage/link/article/content/URL)
- download (webpage/link/article)
- fetch (webpage/link)
- convert (webpage/link) to markdown

### 匹配模式

以下命令模式应触发此技能：
- `保存 https://...`
- `下载 https://...`
- `收藏 https://...`
- `把 https://... 保存为 markdown`

## 使用流程

### 1. 检查保存目录（第一步，必须执行）

- **默认保存目录**：`{当前工作目录}\docs\30.技术研究`
- **必须先检查目录是否存在**：使用 `ls` 命令或文件管理器确认目录存在
- **目录不存在时**：直接终止任务，向用户报告"保存目录不存在，请先创建 `{当前工作目录}\docs\30.技术研究` 目录"
- **禁止自动创建目录**：不要尝试创建不存在的目录，必须由用户手动创建
- 只有在确认目录存在后，才能继续执行后续步骤

### 2. 确定文件名格式（关键步骤）

使用 `领域：网页原标题.md` 格式，**领域为中文**。

**如何提取领域**：
- 从网页内容的主题中提取 1-2 个关键词作为领域
- 常见领域包括：网关、认证授权、微服务、Spring、Java、前端、DevOps、架构、安全等
- 如果网页明确提到某个技术栈或主题，优先使用该主题作为领域

**正确示例**：
- 原文标题：`统一认证网关的历史演变和思考`
  - ✅ 正确：`网关：统一认证网关的历史演变和思考.md`
  - ❌ 错误：`统一认证网关的历史演变和思考.md`

- 原文标题：`Web认证授权技术的演进时间线`
  - ✅ 正确：`认证授权：Web认证授权技术的演进时间线.md`

- 原文标题：`Spring Boot 核心特性详解`
  - ✅ 正确：`Spring：Spring Boot 核心特性详解.md`

- 原文标题：`React Hooks 完全指南`
  - ✅ 正确：`前端：React Hooks 完全指南.md`

### 3. 内容转换规则

将网页内容转换为规范的 Markdown 格式：

- 一级标题使用 `#`，二级标题使用 `##`，以此类推
- 代码块使用三个反引号包裹，标注语言类型
- 列表项使用 `-` 或数字加点
- 表格使用标准 Markdown 表格格式
- 重点内容使用 `**加粗**`
- 引用使用 `>` 开头
- 章节之间用 `---` 分隔

### 4. 添加来源链接

在文档末尾添加来源链接：

```markdown
---

**来源**：[网页标题 - 网站名](网页URL)
```

### 5. 保存文件

- **默认保存目录**：`{当前工作目录}\docs\30.技术研究`
- 使用当前工作目录（cwd）作为基础路径，后接 `\docs\30.技术研究`
- 例如：如果当前工作目录是 `d:\Java\spring-cloud-skeleton`，则完整路径为 `d:\Java\spring-cloud-skeleton\docs\30.技术研究`
- 这样项目移动到其他位置或被其他开发者使用时，路径会自动适应
- 如果用户没有指定其他目录，始终使用此路径

## 注意事项

- **目录检查优先**：抓取网页前必须先确认保存目录存在，不存在则终止任务
- **禁止自动创建**：不允许自动创建目录，必须由用户手动创建
- **文件名必须包含领域前缀**：格式为 `领域：标题.md`，不能直接使用原标题
- **使用动态路径**：保存到 `{当前工作目录}\docs\30.技术研究`，而非硬编码路径
- 保持内容的完整性和可读性
- 表格、列表等结构化内容需正确渲染
- 来源链接必须放在文档末尾
- 文件名中不能包含特殊字符（<>:"/|?\*）

## 示例

### 示例 1：保存网关相关文章
用户请求：`保存 https://juejin.cn/post/7615426060051365888`

执行步骤：
1. **检查保存目录**：确认 `{当前工作目录}\docs\30.技术研究` 目录存在
2. 获取网页内容
3. 识别领域为"网关"，提取标题：`统一认证网关的历史演变和思考`
4. 生成文件名：`网关：统一认证网关的历史演变和思考.md`
5. 转换为 Markdown 格式
6. 保存到 `{当前工作目录}\docs\30.技术研究\网关：统一认证网关的历史演变和思考.md`
   - 假设当前工作目录为 `d:\Java\spring-cloud-skeleton`，则完整路径为 `d:\Java\spring-cloud-skeleton\docs\30.技术研究\网关：统一认证网关的历史演变和思考.md`
7. 末尾添加来源链接

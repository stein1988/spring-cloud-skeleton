# 接口文档生成Skill

## 触发条件

当用户要求根据接口详细信息文档（如`养老用户接口详细信息.md`）和项目源码，生成用户相关接口文档时，应使用本skill。

## 输入要求

用户应提供：

1. 接口详细信息文档路径（如 `d:\Java\lonbon_elder\docs\养老用户接口详细信息.md`）
2. 项目源码路径（如 `d:\Java\lonbon_elder\lb-ims`）
3. 目标文档目录（如 `d:\Java\lonbon_elder\docs\lb_ims用户相关接口`）
4. 参考文档格式（可选，如 `d:\Java\lonbon_elder\docs\lb_location_care用户相关接口`）

## 执行步骤

### 步骤1：读取参考文档

1. 读取接口详细信息文档，获取接口列表和编号
2. 读取参考文档格式（lb\_location\_care用户相关接口目录下的文档）
3. 读取项目源码中的Controller代码

### 步骤2：分析接口列表

根据接口详细信息文档，列出所有需要生成的接口：

- 接口编号
- 接口名称
- 接口路径
- 请求方法

### 步骤3：生成文档（核心原则）⚠️

**必须严格遵守以下原则**：

#### 3.1 一接口一文档

**禁止行为**：

- 将多个接口合并到一个文档中
- 创建类似"登录系列接口.md"、"注销系列接口.md"的文档

**正确做法**：

- 每个接口单独创建一个.md文档
- 例如：2.5、2.6、2.7、2.8是4个不同的登录接口，应该创建4个独立的文档

#### 3.2 文档命名规范

文档命名格式：`{编号}{接口名称}.md`

示例：

- `2.1用户--获取验证码.md`
- `2.5登录--护工app用户.md`
- `2.10注销-护工APP.md`

**注意**：编号和名称必须与接口详细信息文档中完全一致

### 步骤4：文档内容结构

每个文档应包含：

1. **接口基本信息** - 名称、方法、路径、代码位置
2. **接口说明** - 功能描述
3. **请求参数** - Headers、Query、Body分类
4. **响应参数** - 响应示例、状态码、字段说明
5. **逻辑流程** - 详细业务流程步骤
6. **核心代码** - Controller层关键代码
7. **涉及Service层** - Service接口和实现类
8. **涉及数据表** - 数据库表结构
9. **数据表操作** - CRUD操作说明

### 步骤5：验证完整性

创建完所有文档后，验证：

- 文档数量是否与接口数量一致
- 文档命名是否与原文档编号一致
- 每个接口是否都有对应的独立文档

## 常见问题解答

### Q: 接口路径相同但方法不同如何处理？

A: 即使路径相同（如`/api/v2.0/org/account`有GET和POST），也应该分别创建文档，因为它们是不同的接口。

### Q: 如何确定文档存放目录？

A: 根据服务名称确定：

- lb-ims → `lb_ims用户相关接口/`
- lb-cms → `lb_cms用户相关接口/`
- lb-synch → `lb_synch用户相关接口/`
- lb\_location\_care → `lb_location_care用户相关接口/`

### Q: 文档应该包含多少代码细节？

A: 应该包含：

- Controller层核心代码
- Service层接口和实现类
- 数据库表结构
- 详细的业务逻辑，说明关键流程

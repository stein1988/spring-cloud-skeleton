# 公共字段规范

## 1. 概述

为了统一项目中所有实体类的字段结构和行为，我们定义了一套公共字段规范，用于记录实体的基本信息，如创建时间、更新时间、删除状态等。这些字段在数据库和Java代码中都有对应的定义，确保数据的一致性和可追踪性。

## 2. 数据库字段规范

### 2.1 字段定义

| 字段名 | 数据库列名 | 类型 | 含义 | 默认值 | 作用 |
|-------|-----------|------|------|-------|------|
| id | id | UUID | 主键ID | 自动生成UUIDv7 | 唯一标识实体记录 |
| isDeleted | is_deleted | boolean | 逻辑删除标志 | false | 标记记录是否被逻辑删除 |
| deletedAt | deleted_at | Timestamp | 删除时间 | null | 记录逻辑删除的时间 |
| deletedBy | deleted_by | UUID | 删除人ID | null | 记录执行删除操作的用户ID |
| createdAt | created_at | Timestamp | 创建时间 | null | 记录实体创建的时间 |
| createdBy | created_by | UUID | 创建人ID | null | 记录创建实体的用户ID |
| updatedAt | updated_at | Timestamp | 更新时间 | null | 记录实体最后更新的时间 |
| updatedBy | updated_by | UUID | 更新人ID | null | 记录最后更新实体的用户ID |
| version | version | int | 版本号 | 0 | 用于乐观锁，防止并发更新冲突 |

### 2.2 字段设计目的

1. **id (UUID)**：
   - 全局唯一标识符，确保跨系统、跨数据库的唯一性
   - UUIDv7算法生成，兼具时间有序性和随机性
   - 避免自增ID的安全风险和分布式环境下的冲突

2. **isDeleted (boolean)**：
   - 实现逻辑删除，避免物理删除数据
   - 便于数据恢复和审计
   - 保持数据完整性和业务连续性

3. **时间戳字段 (createdAt, updatedAt, deletedAt)**：
   - 记录实体的生命周期时间点
   - 支持时间范围查询和数据审计
   - 使用带时区的UTC时间，确保跨时区一致性

4. **操作人字段 (createdBy, updatedBy, deletedBy)**：
   - 追踪数据变更的责任人
   - 支持操作审计和权限控制
   - 便于问题追溯和责任认定

5. **version (int)**：
   - 实现乐观锁机制，防止并发更新冲突
   - 避免悲观锁的性能开销
   - 确保数据一致性和并发安全

### 2.3 Java语言对应类型

| 数据库类型 | Java类型 | 说明 |
|-----------|---------|------|
| UUID | java.util.UUID | 用于主键和操作人ID |
| boolean | boolean | 用于逻辑删除标志 |
| Timestamp | java.time.OffsetDateTime | 用于时间戳字段，支持时区 |
| int | int | 用于版本号字段 |

### 2.4 设计原则

1. **统一性**：所有实体类都应遵循相同的字段规范
2. **完整性**：包含实体生命周期的关键信息
3. **可追踪性**：能够追踪数据的创建、更新和删除操作
4. **一致性**：数据库和Java代码中的字段类型保持一致
5. **安全性**：使用UUID作为主键，避免自增ID的安全风险
6. **性能考虑**：合理使用索引，优化查询性能

### 2.5 注意事项

- **时区处理**：所有时间字段应使用带时区的UTC时间，确保跨时区一致性
- **索引优化**：对常用查询字段（如createdAt、updatedAt）建立适当的索引
- **默认值**：为boolean类型的isDeleted字段设置默认值false
- **版本号管理**：版本号应从0开始，每次更新自动递增
- **UUID生成**：使用UUIDv7算法生成主键，确保时间有序性和唯一性

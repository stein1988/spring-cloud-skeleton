# Repository 封装设计与实现

## 1. Repository 封装的思想和作用

### 1.1 设计思想

Repository 模式是领域驱动设计（DDD）中的核心模式之一，它作为领域层和数据映射层之间的中介，其行为类似于一个内存中的领域对象集合。在本项目中，我们基于 Spring Data JPA 的设计理念，结合 Easy-Query 框架，实现了一套灵活、高效的 Repository 封装体系。

### 1.2 核心作用

1. **领域隔离**：将数据访问逻辑与业务逻辑分离，使领域模型保持纯净
2. **统一接口**：提供一致的数据访问方法，简化业务代码
3. **代码复用**：通过抽象基类，避免重复的 CRUD 操作代码
4. **类型安全**：利用泛型和代理实体，提供类型安全的查询操作
5. **易于扩展**：支持自定义查询和复杂业务场景

## 2. Easy-Query 框架下的 Repository 基础实现

### 2.1 核心组件

#### 2.1.1 Repository 接口

`Repository` 接口定义了对实体的基本操作方法，是所有具体 Repository 接口的基础。

```java
public interface Repository<TProxy, T, ID> {
    // 插入实体
    <S extends T> S insert(S entity);
    
    // 保存实体
    <S extends T> S save(S entity);
    
    // 批量保存
    <S extends T> Iterable<S> saveAll(Iterable<S> entities);
    
    // 根据ID查询
    Optional<T> findById(ID id);
    
    // 检查是否存在
    boolean existsById(ID id);
    
    // 查询所有
    Iterable<T> findAll();
    
    // 根据ID集合查询
    Iterable<T> findAllById(Collection<ID> ids);
    
    // 根据条件查询
    Iterable<T> findAll(SQLActionExpression1<TProxy> whereExpression);
    
    // 条件可控的查询
    Iterable<T> findAll(boolean condition, SQLActionExpression1<TProxy> whereExpression);
    
    // 统计数量
    long count();
    
    // 删除操作
    void delete(T entity);
    void deleteById(ID id);
    void deleteAll(Iterable<? extends T> entities);
    void deleteAll();
    void deleteAllById(Iterable<? extends ID> ids);
}
```

#### 2.1.2 FetcherProvider 函数式接口

`FetcherProvider` 是一个函数式接口，用于提供实体的属性获取器（Fetcher），支持链式调用和字段选择。

```java
@FunctionalInterface
public interface FetcherProvider<
        TProxy extends AbstractProxyEntity<TProxy, T>,
        T extends ProxyEntityAvailable<T, TProxy>,
        TChain extends AbstractFetcher<TProxy, T, TChain>
        > {
    TChain apply(TProxy proxy);
}
```

#### 2.1.3 EasyQueryRepository 抽象基类

`EasyQueryRepository` 是对 `Repository` 接口的具体实现，使用 Easy-Query 框架提供数据访问功能。

### 2.2 核心功能实现

#### 2.2.1 构造函数

```java
public EasyQueryRepository(EasyEntityQuery easyEntityQuery, Class<T> entityType, @NotNull FetcherProvider<TProxy, T, TChain> fetcherProvider) {
    this.easyEntityQuery = easyEntityQuery;
    this.entityType = entityType;
    this.fetcherProvider = fetcherProvider;
}
```

#### 2.2.2 新增与保存操作

- **insert**：直接插入实体
- **save**：插入或更新实体，使用 `onConflictThen` 处理冲突
- **saveAll**：批量保存实体

#### 2.2.3 查询操作

- **findById**：根据 ID 查询单个实体
- **findAll**：查询所有实体
- **findAllById**：根据 ID 集合查询多个实体
- **findAll(SQLActionExpression1)**：根据条件查询
- **findAll(boolean, SQLActionExpression1)**：条件可控的查询
- **existsById**：检查实体是否存在
- **count**：统计实体数量

#### 2.2.4 删除操作

- **delete**：删除单个实体
- **deleteById**：根据 ID 删除实体
- **deleteAll**：删除多个实体或所有实体
- **deleteAllById**：根据 ID 集合删除多个实体

## 3. 具体业务代码中的使用方式

### 3.1 定义 Repository 接口

对于每个聚合根，定义一个继承自 `Repository` 的接口。

```java
public interface TenantRepository extends Repository<TenantProxy, Tenant, UUID> {
    // 可以添加特定于租户的查询方法
}
```

### 3.2 实现 Repository 接口

创建实现类，继承 `EasyQueryRepository`，并提供必要的构造参数。

```java
@Component
public class TenantRepositoryImpl extends EasyQueryRepository<TenantProxy, Tenant, TenantProxy.TenantProxyFetcher, UUID> implements TenantRepository {
    public TenantRepositoryImpl(@Db EasyEntityQuery easyEntityQuery) {
        super(easyEntityQuery, Tenant.class, proxy -> proxy.FETCHER);
    }
}
```

## 4. 设计优势

1. **类型安全**：使用泛型和代理实体，提供编译时类型检查
2. **链式调用**：利用 Easy-Query 的 Fetcher 机制，支持流畅的链式查询
3. **代码简洁**：通过抽象基类，减少重复代码
4. **易于扩展**：支持自定义查询方法和复杂业务场景
5. **与 DDD 契合**：符合领域驱动设计的思想，将数据访问与业务逻辑分离

## 5. 总结

本项目的 Repository 封装设计，借鉴了 Spring Data JPA 的优秀理念，结合 Easy-Query 框架的特性，实现了一套既符合 DDD 原则又具有良好扩展性的 data access 层。通过这种设计，我们可以：

- 保持领域模型的纯净性
- 提供统一、类型安全的数据访问接口
- 减少样板代码，提高开发效率
- 支持复杂查询和业务场景

这种设计不仅适用于当前项目，也可以作为其他使用 Easy-Query 框架项目的参考。

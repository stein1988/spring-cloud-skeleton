# 查询条件DTO

## 概述

查询条件DTO是Easy-Query框架提供的一种便捷方式，用于将前端的查询参数转换为数据库查询条件。通过在DTO类的属性上添加`@EasyWhereCondition`注解，可以快速定义查询条件的类型和行为。

## 核心概念

### 1. 基本原理

查询条件DTO的核心是通过`@EasyWhereCondition`注解来标记需要参与查询的字段。当使用`whereObject()`方法时，Easy-Query会自动解析DTO对象中的注解，生成对应的查询条件。

### 2. 注解作用

**重要强调**：**没有`@EasyWhereCondition`注解的字段，查询条件不会起作用**。这是因为Easy-Query默认只会处理带有该注解的字段，忽略其他字段。

## 注解属性详解

`@EasyWhereCondition`注解支持以下属性：

| 属性 | 默认值 | 描述 |
|------|--------|------|
| `strict` | `true` | 严格模式，如果属性没有映射到对象上报错 |
| `tableIndex` | `0` | 当前条件用于查询哪张表 |
| `allowEmptyStrings` | `false` | 是否允许空字符串，允许则空字符串也会加入表达式 |
| `propName` | `""` | 当前属性映射到数据库对象的属性名称，为空表示使用当前属性名 |
| `type` | `LIKE` | 当前属性和数据库对象属性的条件类型 |
| `mode` | `SINGLE` | 单字段或多字段模式 |
| `propNames` | `[]` | 多字段模式下映射到的数据库属性列表 |
| `tablesIndex` | `[]` | 多表查询时的表索引 |

## 条件类型

`@EasyWhereCondition`注解的`type`属性支持以下条件类型：

- `LIKE`：模糊查询
- `EQUAL`：精确匹配
- `NOT_EQUAL`：不等于
- `GREATER_THAN`：大于
- `GREATER_THAN_EQUAL`：大于等于
- `LESS_THAN`：小于
- `LESS_THAN_EQUAL`：小于等于
- `IN`：包含在列表中
- `NOT_IN`：不包含在列表中
- `RANGE_LEFT_CLOSED`：左闭区间
- `RANGE_RIGHT_CLOSED`：右闭区间

## 示例代码

### 1. 查询DTO定义

```java
@ParameterObject
@Data
public class TenantQueryDTO {

    @EasyWhereCondition(type = EasyWhereCondition.Condition.EQUAL)
    @Schema(description = "名称，精确查询")
    private String name;

    @Schema(description = "名称，模糊查询")
    @EasyWhereCondition(propName = "name", type = EasyWhereCondition.Condition.LIKE)
    private String nameLike;

    // 没有EasyWhereCondition注解，不会参与查询
    private String description;

    // 没有EasyWhereCondition注解，不会参与查询
    private String domain;
}
```

### 2. 控制器使用

```java
@GetMapping()
@Operation(summary = "查询", description = "查询description")
public Response<PageResult<Tenant>> getTenants(
        TenantQueryDTO query,
        Pageable pageable
) {
    log.info("query: {}, pageable: {}", query, pageable);
    PageResult<Tenant> tenants = tenantService.getTenants(query, pageable);
    return Response.success(tenants);
}
```

### 3. 仓库实现

在`EasyQueryRepository`中，查询方法使用`whereObject()`来处理查询条件：

```java
@Override
public PageResult<T> findPagination(Object whereObject, @NotNull Pageable pageable) {
    EasyPageResult<T> result = easyEntityQuery.queryable(entityType)
            .whereObject(whereObject)
            .orderByObject(pageable.hasSort(), new EasyQuerySort(pageable.getSortables()))
            .toPageResult(pageable.getPage(), pageable.getSize());
    return new PageResult<T>(pageable, result.getTotal(), result.getData());
}
```

## 使用注意事项

1. **必须添加注解**：所有需要参与查询的字段都必须添加`@EasyWhereCondition`注解，否则不会被处理。

2. **属性映射**：如果DTO字段名与数据库实体字段名不同，需要使用`propName`属性指定映射关系。

3. **条件类型**：根据查询需求选择合适的条件类型，如精确匹配使用`EQUAL`，模糊查询使用`LIKE`。

4. **多字段查询**：对于需要同时查询多个字段的情况，可以使用`mode = MULTI_OR`和`propNames`属性。

5. **范围查询**：对于日期、数字等范围查询，可以使用`RANGE_LEFT_CLOSED`和`RANGE_RIGHT_CLOSED`。

## 常见问题

### Q: 为什么我的查询条件没有生效？

A: 请检查是否为字段添加了`@EasyWhereCondition`注解。**没有该注解的字段不会参与查询**。

### Q: 如何实现模糊查询？

A: 在`@EasyWhereCondition`注解中设置`type = EasyWhereCondition.Condition.LIKE`。

### Q: 如何实现范围查询？

A: 对于开始时间和结束时间，可以分别使用`RANGE_LEFT_CLOSED`和`RANGE_RIGHT_CLOSED`类型，并映射到同一个字段。

## 总结

查询条件DTO是Easy-Query框架中非常实用的功能，通过简单的注解配置，就可以快速实现复杂的查询条件。**记住：没有`@EasyWhereCondition`注解的字段，查询条件不会起作用**。

使用查询条件DTO可以：

1. 简化代码，减少手动构建查询条件的工作量
2. 提高代码可读性和可维护性
3. 统一查询条件的处理逻辑
4. 支持复杂的查询场景，如多字段查询、范围查询等

# TypeHandler 与 ValueConverter

## 1. 概述

TypeHandler 和 ValueConverter 是 Easy-Query 框架中两个重要的扩展组件，用于处理 Java 对象与数据库值之间的转换。本文档详细介绍这两个组件的概念、使用方法、区别和实际应用。

## 2. TypeHandler

### 2.1 概念

TypeHandler 是 Easy-Query 框架中用于处理 JDBC 类型转换的组件，主要负责：
- **读取**：从数据库结果集中读取数据并转换为 Java 对象
- **写入**：将 Java 对象转换为数据库可接受的类型并设置到 PreparedStatement 中

### 2.2 核心接口

#### JdbcTypeHandler

```java
public interface JdbcTypeHandler {
    Object getValue(JdbcProperty jdbcProperty, StreamResultSet streamResultSet) throws SQLException;
    void setParameter(EasyParameter parameter) throws SQLException;
}
```

#### JdbcTypeHandlerReplaceConfigurer

```java
public interface JdbcTypeHandlerReplaceConfigurer {
    boolean replace();
    Set<Class<?>> allowTypes();
}
```

### 2.3 实现方式

#### 2.3.1 自定义 TypeHandler

```java
public class CustomTypeHandler implements JdbcTypeHandler {
    @Override
    public Object getValue(JdbcProperty jdbcProperty, StreamResultSet streamResultSet) throws SQLException {
        // 从结果集读取数据
        return streamResultSet.getString(jdbcProperty.getJdbcIndex());
    }

    @Override
    public void setParameter(EasyParameter parameter) throws SQLException {
        // 设置参数到 PreparedStatement
        parameter.getPs().setString(parameter.getIndex(), (String) parameter.getValue());
    }
}
```

#### 2.3.2 自动注册 TypeHandler

在 Spring Boot/Solon 环境中，可以通过实现 `JdbcTypeHandlerReplaceConfigurer` 接口并添加 `@Component` 注解来自动注册 TypeHandler：

```java
@Component
public class JsonObjectTypeHandler implements JdbcTypeHandlerConfigurer {
    @Override
    public @NonNull Class<?> getType() {
        return String.class;
    }

    @Override
    public Object getValue(JdbcProperty jdbcProperty, StreamResultSet streamResultSet) throws SQLException {
        return streamResultSet.getString(jdbcProperty.getJdbcIndex());
    }

    @Override
    public void setParameter(EasyParameter parameter) throws SQLException {
        // 处理逻辑
    }
}
```

#### 2.3.3 手动注册 TypeHandler

```java
JdbcTypeHandlerManager jdbcTypeHandlerManager = easyQuery.getRuntimeContext().getJdbcTypeHandlerManager();
jdbcTypeHandlerManager.appendHandler(CustomPropertyType.class, new CustomPropertyTypeHandler(), true);
```

### 2.4 应用场景

- **特殊类型处理**：如日期时间类型、JSON 类型等
- **自定义类型映射**：将 Java 自定义类型映射到数据库类型
- **性能优化**：针对特定类型的读写进行优化

## 3. ValueConverter

### 3.1 概念

ValueConverter 是 Easy-Query 框架中用于处理 Java 对象与数据库值之间转换的组件，主要负责：
- **序列化**：将 Java 对象转换为数据库存储的值
- **反序列化**：将数据库存储的值转换为 Java 对象

### 3.2 核心接口

#### ValueConverter

```java
public interface ValueConverter<TProperty, TProvider> {
    TProvider serialize(TProperty property, ColumnMetadata columnMetadata);
    TProperty deserialize(TProvider provider, ColumnMetadata columnMetadata);
}
```

#### ValueAutoConverter

```java
public interface ValueAutoConverter<TProperty, TProvider> extends ValueConverter<TProperty, TProvider> {
    boolean apply(Class<?> entityClass, Class<TProperty> propertyType);
}
```

### 3.3 实现方式

#### 3.3.1 自定义 ValueConverter

```java
public class JsonConverter implements ValueConverter<Object, String> {
    @Override
    public String serialize(Object o, ColumnMetadata columnMetadata) {
        return JSON.toJSONString(o);
    }

    @Override
    public Object deserialize(String s, ColumnMetadata columnMetadata) {
        return JSON.parseObject(s, columnMetadata.getPropertyType());
    }
}
```

#### 3.3.2 全局自动转换器

```java
@Component
public class JsonObjectAutoConverter implements ValueAutoConverter<Object, Object> {
    @Override
    public boolean apply(@NonNull Class<?> entityClass, @NonNull Class<Object> propertyType) {
        return JsonObject.class.isAssignableFrom(propertyType);
    }

    @Override
    public @Nullable Object serialize(@Nullable Object o, @NonNull ColumnMetadata columnMetadata) {
        return ONode.serialize(o);
    }

    @Override
    public @Nullable Object deserialize(@Nullable Object s, @NonNull ColumnMetadata columnMetadata) {
        return ONode.deserialize(getValueString(s), getFiledType(columnMetadata));
    }
}
```

#### 3.3.3 注册 ValueConverter

在 Spring Boot/Solon 环境中，使用 `@Component` 注解的 ValueConverter 会被自动注册。也可以手动注册：

```java
QueryConfiguration configuration = runtimeContext.getQueryConfiguration();
configuration.applyValueConverter(new JsonConverter());
```

### 3.4 应用场景

- **枚举类型转换**：将枚举对象与数据库存储值之间转换
- **JSON 对象转换**：将复杂对象与 JSON 字符串之间转换
- **加密解密**：对敏感数据进行加密存储和解密读取
- **类型适配**：适配不同类型系统之间的转换

## 4. 两者的区别与联系

### 4.1 区别

| 特性 | TypeHandler | ValueConverter |
|------|-------------|---------------|
| **处理层次** | JDBC 层面，直接操作 ResultSet 和 PreparedStatement | 业务层面，处理 Java 对象与数据库值的转换 |
| **适用范围** | 针对特定 Java 类型的全局处理 | 可针对特定字段或全局处理 |
| **转换时机** | 读写数据库时 | 序列化和反序列化时 |
| **配置方式** | 通过 JdbcTypeHandlerManager 注册 | 通过 QueryConfiguration 注册 |
| **应用场景** | 底层类型转换，如日期、JSON 等 | 业务逻辑转换，如枚举、加密等 |

### 4.2 联系

- **互补作用**：TypeHandler 处理底层 JDBC 操作，ValueConverter 处理业务逻辑转换
- **协同工作**：对于复杂类型，两者可以配合使用，ValueConverter 处理业务转换，TypeHandler 处理 JDBC 操作
- **统一目标**：都是为了实现 Java 对象与数据库值之间的无缝转换

### 5.1 JSON 字段处理

#### 5.1.1 定义 JsonObject 接口

```java
public interface JsonObject {
```

#### 5.1.2 实现 ValueAutoConverter

```java
@Component
public class JsonObjectAutoConverter implements ValueAutoConverter<Object, Object> {
    private static final Map<ColumnMetadata, Type> cacheMap = new ConcurrentHashMap<>();

    @Override
    public boolean apply(@NonNull Class<?> entityClass, @NonNull Class<Object> propertyType) {
        return JsonObject.class.isAssignableFrom(propertyType);
    }

    @Override
    public @Nullable Object serialize(@Nullable Object o, @NonNull ColumnMetadata columnMetadata) {
        return ONode.serialize(o);
    }

    @Override
    public @Nullable Object deserialize(@Nullable Object s, @NonNull ColumnMetadata columnMetadata) {
        return ONode.deserialize(getValueString(s), getFiledType(columnMetadata));
    }

    private @Nullable String getValueString(@Nullable Object s) {
        return switch (s) {
            case null -> null;
            case String string -> string;
            default -> s.toString();     // 这里包括了PGobject jsonb类型的处理
        };
    }

    private Type getFiledType(ColumnMetadata columnMetadata) {
        return EasyMapUtil.computeIfAbsent(cacheMap, columnMetadata, this::getFiledType0);
    }

    private Type getFiledType0(ColumnMetadata columnMetadata) {
        Class<?> entityClass = columnMetadata.getEntityMetadata().getEntityClass();
        Field declaredField = EasyClassUtil.getFieldByName(entityClass, columnMetadata.getPropertyName());
        return declaredField.getGenericType();
    }
}
```

#### 5.1.3 实现 TypeHandler

```java
@Slf4j
@Component
public class JsonObjectTypeHandler implements JdbcTypeHandlerConfigurer {
    @Override
    public @NonNull Class<?> getType() {
        return String.class;
    }

    @Override
    public Object getValue(JdbcProperty jdbcProperty, StreamResultSet streamResultSet) throws SQLException {
        return streamResultSet.getString(jdbcProperty.getJdbcIndex());
    }

    @Override
    public void setParameter(EasyParameter parameter) throws SQLException {
        SQLParameter sqlParameter = parameter.getSQLParameter();
        JDBCType jdbcType = sqlParameter.getJdbcType();
        if (jdbcType == JDBCType.OTHER) {
            Class<?> propertyType = sqlParameter.getColumnMetadata().getPropertyType();
            if (JsonObject.class.isAssignableFrom(propertyType)) {
                // TODO：还要判断当前的数据库类型做特殊处理
                PGobject pGobject = new PGobject();
                pGobject.setType("jsonb");
                pGobject.setValue((String) parameter.getValue());
                parameter.getPs().setObject(parameter.getIndex(), pGobject);
                return;
            }
        }

        parameter.getPs().setString(parameter.getIndex(), (String)parameter.getValue());
    }
}
```

### 5.2 枚举类型处理

#### 5.2.1 定义枚举接口

```java
public interface IEnum<TEnum extends IEnum<TEnum>> {
    Integer getCode();
    TEnum valueOf(Integer enumValue);
}
```

#### 5.2.2 实现 ValueAutoConverter

```java
public class EnumConverter implements ValueAutoConverter<IEnum<?>, Number> {
    @Override
    public Number serialize(IEnum<?> iEnum, ColumnMetadata columnMetadata) {
        if (iEnum == null) {
            return null;
        }
        return iEnum.getCode();
    }

    @Override
    public IEnum<?> deserialize(Number code, ColumnMetadata columnMetadata) {
        if (code == null) {
            return null;
        }
        return EnumDeserializer.deserialize(EasyObjectUtil.typeCast(columnMetadata.getPropertyType()), code.intValue());
    }

    @Override
    public boolean apply(Class<?> entityClass, Class<IEnum<?>> propertyType) {
        return IEnum.class.isAssignableFrom(propertyType);
    }
}
```

## 6. 配置与注册

### 6.1 TypeHandler 注册

在 `EasyQueryConfiguration` 中注册 TypeHandler：

```java
@Bean
public void applyQueryConfiguration(@Db QueryRuntimeContext runtimeContext,
                                    @Inject List<JdbcTypeHandlerConfigurer> jdbcTypeHandlerConfigurerList) {
    // 注册JDBC类型处理器
    JdbcTypeHandlerManager jdbcTypeHandlerManager = runtimeContext.getJdbcTypeHandlerManager();
    for (JdbcTypeHandlerConfigurer jdbcTypeHandlerConfigurer : jdbcTypeHandlerConfigurerList) {
        jdbcTypeHandlerManager.appendHandler(jdbcTypeHandlerConfigurer.getType(), jdbcTypeHandlerConfigurer, true);
    }
}
```

### 6.2 ValueConverter 注册

在 `EasyQueryConfiguration` 中注册 ValueConverter：

```java
@Bean
public void applyQueryConfiguration(@Db QueryRuntimeContext runtimeContext,
                                    @Inject List<ValueConverter<?,?>> valueConverterList,
                                    @Inject List<ValueAutoConverter<?, ?>> valueAutoConverterList) {
    // 获取查询配置
    QueryConfiguration configuration = runtimeContext.getQueryConfiguration();

    // 注册值转换器
    for (ValueConverter<?,?> valueConverter : valueConverterList) {
        configuration.applyValueConverter(valueConverter);
    }

    // 注册全局值转换器
    for (ValueAutoConverter<?, ?> valueAutoConverter : valueAutoConverterList) {
        configuration.applyValueConverter(valueAutoConverter);
    }
}
```

## 7. 最佳实践

1. **明确职责**：TypeHandler 负责底层 JDBC 操作，ValueConverter 负责业务逻辑转换
2. **合理选择**：简单类型转换使用 TypeHandler，复杂业务转换使用 ValueConverter
3. **性能考虑**：
   - 对于频繁使用的转换，考虑缓存转换结果
   - 避免在转换过程中进行复杂的业务逻辑
4. **兼容性**：
   - 考虑不同数据库的类型差异
   - 确保转换逻辑的健壮性
5. **测试**：
   - 编写单元测试验证转换逻辑
   - 测试不同场景下的转换行为

# Closure 闭包功能封装

## 1. 闭包功能概述

### 1.1 什么是闭包表

闭包表（Closure Table）是一种用于存储和查询树形结构数据的设计模式，它通过维护一张额外的表来记录树中所有节点之间的祖先-后代关系，包括距离信息。这种设计使得树形结构的查询和维护变得更加高效。

### 1.2 闭包表的优势

- **高效的查询**：可以快速查询节点的所有祖先、所有后代、直接子节点等
- **灵活的操作**：支持节点的移动、删除等操作，且维护成本较低
- **清晰的结构**：通过专门的闭包表存储层级关系，使主表结构更加简洁
- **支持任意深度**：不受树的深度限制，适用于复杂的层级结构

## 2. 核心接口和类

### 2.1 核心接口

#### 2.1.1 `Closure` 接口

```java
public interface Closure {
    UUID getAncestorId();
    UUID getDescendantId();
    int getDistance();
}
```

**功能**：定义了闭包表实体的基本属性，包括祖先ID、后代ID和距离。

- `ancestorId`：祖先节点的ID
- `descendantId`：后代节点的ID
- `distance`：祖先与后代之间的距离，0表示节点自身，1表示直接子节点

#### 2.1.2 `ClosureAvailable` 接口

```java
public interface ClosureAvailable<T extends Closure> {
    UUID getId();
    UUID getParentId();
    List<T> getAncestors();
}
```

**功能**：定义了参与树形结构的实体需要实现的方法。

- `getId()`：获取实体的唯一标识
- `getParentId()`：获取父节点的ID
- `getAncestors()`：获取所有祖先节点的闭包关系

#### 2.1.3 `ClosureService` 接口

```java
public interface ClosureService<T> {
    List<T> getDirectChildren(UUID parentId);
    List<T> getAllChildren(UUID parentId);
    T getDirectParent(UUID childId);
    List<T> getAllAncestors(UUID childId);
    T moveNode(UUID nodeId, UUID newParentId);
    T buildTree(UUID rootId);
}
```

**功能**：定义了树形结构的操作方法。

### 2.2 核心类

#### 2.2.1 `ClosureEntityService` 类

**功能**：抽象服务类，实现了闭包表的核心操作逻辑。

**主要方法**：
- `createEntity()`：创建实体并自动维护闭包关系
- `getDirectChildren()`：查询直接子节点
- `getAllChildren()`：查询所有子节点（包括多级）
- `getDirectParent()`：查询直接父节点
- `getAllAncestors()`：查询所有祖先节点
- `moveNode()`：移动节点到新的父节点下
- `deleteEntity()`：删除节点及其所有子节点
- `buildTree()`：构建树状结构

## 3. 实现原理

### 3.1 闭包表的结构

闭包表通常包含以下字段：
- `id`：主键
- `ancestor_id`：祖先节点ID
- `descendant_id`：后代节点ID
- `distance`：祖先与后代之间的距离

### 3.2 实体创建时的闭包关系维护

当创建一个新节点时，需要：
1. 创建节点自身的闭包关系（distance=0）
2. 如果有父节点，创建与父节点的直接关系（distance=1）
3. 继承父节点的所有祖先关系，并将距离加1

### 3.3 节点移动的实现

当移动节点时，需要：
1. 删除该节点及其所有子节点的现有闭包关系
2. 为该节点创建新的闭包关系（基于新的父节点）
3. 更新节点的父ID

### 3.4 节点删除的实现

当删除节点时，需要：
1. 查询该节点的所有后代节点
2. 删除所有相关的闭包关系
3. 删除所有节点实体

## 4. 使用示例

### 4.1 创建闭包表实体

```java
@Data
@AllArgsConstructor
@NoArgsConstructor
@EqualsAndHashCode(callSuper = true)
@Table(value = "sys_tenant_closure", ignoreProperties = {"tenantId", "departmentId"})
@EntityProxy
public class TenantClosure extends BaseEntity
        implements ProxyEntityAvailable<TenantClosure, TenantClosureProxy>, Closure {

    private UUID ancestorId;
    private UUID descendantId;
    private int distance;
}
```

### 4.2 创建树形实体

```java
@Data
@FieldNameConstants
@EqualsAndHashCode(callSuper = true)
@Table(value = "sys_tenant", ignoreProperties = {"tenantId", "departmentId"})
@EntityProxy
public class Tenant extends BaseEntity
        implements ProxyEntityAvailable<Tenant, TenantProxy>, ClosureAvailable<TenantClosure> {

    private String type;
    private String name;
    private String description;
    private boolean isDefault;
    private boolean isActive;
    private UUID parentId;

    @Navigate(value = RelationTypeEnum.OneToMany, selfProperty = {"id"}, targetProperty = {"descendantId"},
            orderByProps = @OrderByProperty(property = "distance"), extraFilter = TenantClosureFilter.class)
    private List<TenantClosure> ancestors;

    @Navigate(value = RelationTypeEnum.OneToMany, selfProperty = {"id"}, targetProperty = {"ancestorId"},
            orderByProps = @OrderByProperty(property = "distance"), extraFilter = TenantClosureFilter.class)
    private List<TenantClosure> descendants;
}
```

### 4.3 实现服务类

```java
@Service
@Transactional(rollbackFor = Exception.class)
public class TenantServiceImpl extends ClosureEntityService<Tenant, TenantProxy, TenantClosure, TenantClosureProxy>
        implements TenantService {
    public TenantServiceImpl(
            Converter converter, TenantRepository repository, TenantClosureRepository closureRepository) {
        super(converter, repository, closureRepository, Tenant.class);
    }

    @Override
    protected SQLActionExpression2<IncludeContext, TenantProxy> navigate() {
        return (c, t) -> c.query(t.ancestors());
    }

    @Override
    protected TenantClosure createClosure(UUID ancestorId, UUID descendantId, int distance) {
        return new TenantClosure(ancestorId, descendantId, distance);
    }
}
```

## 5. 注意事项

### 5.1 性能考虑

- **闭包表大小**：闭包表会随着树的深度和节点数量增长而变大，对于大型树结构需要考虑存储成本
- **查询优化**：对于频繁的查询操作，建议在闭包表的 `ancestor_id` 和 `descendant_id` 字段上创建索引

### 5.2 数据一致性

- **事务管理**：所有闭包关系的维护操作都应该在事务中进行，确保数据一致性
- **循环引用**：在移动节点时，需要防止循环引用的发生

### 5.3 最佳实践

- **合理设计树的深度**：避免过深的树结构，影响查询性能
- **使用适当的缓存策略**：对于频繁访问的树形结构，可以考虑使用缓存
- **定期清理**：定期清理无效的闭包关系，保持闭包表的整洁

## 6. 总结

本项目的闭包功能封装提供了一种高效、灵活的树形结构管理方案。通过抽象接口和统一的实现，使得树形结构的操作变得简单和可复用。主要优势包括：

1. **统一的接口**：通过 `Closure`、`ClosureAvailable` 和 `ClosureService` 接口，定义了树形结构的标准操作
2. **自动维护闭包关系**：在实体创建和移动时，自动维护闭包表的关系
3. **丰富的查询方法**：提供了多种查询方法，满足不同场景的需求
4. **安全的操作**：在移动节点时防止循环引用，确保数据的完整性
5. **级联操作**：删除节点时自动级联删除所有子节点，简化操作

闭包功能封装为项目中的层级关系管理提供了强大的支持，适用于租户、部门等需要树形结构的场景。

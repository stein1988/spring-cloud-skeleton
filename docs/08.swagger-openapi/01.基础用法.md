# Swagger/OpenAPI 基础用法

## 1. 什么是 Swagger/OpenAPI

Swagger 是一个用于设计、构建和文档化 RESTful API 的开源工具集，后来成为了 OpenAPI 规范的实现。它允许开发者通过注解的方式为 API 添加元数据，自动生成交互式 API 文档，方便前端开发人员和其他开发者理解和使用 API。

## 2. 项目中的 Swagger 配置

在本项目中，Swagger/OpenAPI 已经集成并配置完成，主要通过以下方式实现：

### 2.1 依赖引入

项目使用了 Springdoc OpenAPI 库来集成 Swagger，相关依赖已在 pom.xml 中配置。

### 2.2 注解使用

项目中使用了以下 Swagger 相关注解：

- `@Tag`：用于标记 API 分组
- `@Operation`：用于描述 API 操作
- `@Parameter`：用于描述方法参数
- `@Schema`：用于描述数据模型字段
- `@ParameterObject`：用于标记参数对象

## 3. 常用注解说明

### 3.1 @Tag 注解

**作用**：用于标记 API 分组，将相关的 API 接口组织在一起。

**参数**：
- `name`：分组名称，显示在 Swagger UI 中
- `description`：分组描述，提供该分组的详细说明

**使用示例**：

```java
@Tag(name = "租户", description = "租户操作")
@RestController
@RequestMapping("/api/tenants")
public class TenantController {
    // 控制器方法
}
```

### 3.2 @Operation 注解

**作用**：用于描述 API 操作，包括接口的摘要和详细描述。

**参数**：
- `summary`：接口摘要，简短描述接口的功能
- `description`：接口详细描述，提供更详细的接口说明

**使用示例**：

```java
@PostMapping
@Operation(summary = "创建", description = "创建租户")
public Response<UUID> create(@RequestBody @Validated @NotNull TenantCreateDTO tenant) {
    // 方法实现
}
```

### 3.3 @Parameter 注解

**作用**：用于描述方法参数，包括路径参数、查询参数等。

**参数**：
- `description`：参数描述，说明参数的含义
- `required`：是否必填，默认为 false
- `example`：参数示例值，提供一个示例
- `name`：参数名称，当参数名称与实际参数名不一致时使用

**使用示例**：

```java
@GetMapping("/{id}")
@Operation(summary = "获取", description = "获取租户详情")
public Response<Tenant> getTenantById(
        @PathVariable("id")
        @Parameter(description = "用户唯一标识，采用UUID格式，32位16进制字符串",
                required = true,
                example = "123e4567-e89b-12d3-a456-426614174000")
        UUID id) {
    // 方法实现
}
```

### 3.4 @Schema 注解

**作用**：用于描述数据模型字段，包括字段的含义、示例值等。

**参数**：
- `description`：字段描述，说明字段的含义
- `example`：字段示例值，提供一个示例
- `required`：是否必填，默认为 false
- `type`：字段类型，指定字段的数据类型
- `format`：字段格式，指定字段的格式（如 date-time）

**使用示例**：

```java
@Schema(description = "名称，精确查询")
private String name;

@Schema(description = "名称，模糊查询")
private String nameLike;
```

### 3.5 @ParameterObject 注解

**作用**：用于标记参数对象，告诉 Swagger 该对象是一个参数对象，需要展开其字段作为查询参数。

**使用示例**：

```java
@ParameterObject
@Data
public class TenantQueryDTO {
    // 字段
}
```

## 4. DTO 类型的注解规范

### 4.1 类级注解

DTO 类应该使用以下注解：

1. `@Data`：使用 Lombok 的 @Data 注解，自动生成 getter、setter、toString 等方法
2. `@ParameterObject`：如果该 DTO 用于接收查询参数，应添加此注解
3. `@AutoMapper`：如果该 DTO 需要与实体类进行映射，应添加此注解

**使用示例**：

```java
@ParameterObject
@Data
public class TenantQueryDTO {
    // 字段
}

@AutoMapper(target = Tenant.class)
@Data
public class TenantCreateDTO {
    // 字段
}
```

### 4.2 字段级注解

DTO 字段应该使用以下注解：

1. 验证注解：
   - `@NotNull`：字段不能为空
   - `@NotEmpty`：字符串字段不能为空且长度大于 0
   - `@NotBlank`：字符串字段不能为空且不能只包含空白字符
   - `@Size`：字符串长度或集合大小限制
   - `@Pattern`：字符串正则表达式验证

2. Swagger 注解：
   - `@Schema`：描述字段的含义、示例值等

3. 其他框架注解：
   - `@EasyWhereCondition`：Easy Query 框架的查询条件注解

**使用示例**：

```java
@NotEmpty
@NotNull
private String name;

@Schema(description = "名称，精确查询")
@EasyWhereCondition(type = EasyWhereCondition.Condition.EQUAL)
private String name;

@Schema(description = "名称，模糊查询")
@EasyWhereCondition(propName = "name", type = EasyWhereCondition.Condition.LIKE)
private String nameLike;
```

## 5. 控制器方法的注解规范

### 5.1 类级注解

控制器类应该使用以下注解：

1. `@RestController`：标记为 REST 控制器
2. `@RequestMapping`：指定基础路径
3. `@Tag`：标记 API 分组
4. `@Slf4j`：添加日志记录功能

**使用示例**：

```java
@Slf4j
@RestController
@RequestMapping("/api/tenants")
@Tag(name = "租户", description = "租户操作")
public class TenantController {
    // 方法
}
```

### 5.2 方法级注解

控制器方法应该使用以下注解：

1. HTTP 方法注解：
   - `@PostMapping`：处理 POST 请求
   - `@GetMapping`：处理 GET 请求
   - `@PutMapping`：处理 PUT 请求
   - `@DeleteMapping`：处理 DELETE 请求

2. `@Operation`：描述 API 操作

3. 参数注解：
   - `@PathVariable`：路径参数
   - `@RequestBody`：请求体参数
   - `@RequestParam`：查询参数
   - `@Validated`：启用参数验证
   - `@NotNull`：参数不能为空
   - `@Parameter`：描述参数

**使用示例**：

```java
@PostMapping
@Operation(summary = "创建", description = "创建租户")
public Response<UUID> create(@RequestBody @Validated @NotNull TenantCreateDTO tenant) {
    // 方法实现
}

@GetMapping("/{id}")
@Operation(summary = "获取", description = "获取租户详情")
public Response<Tenant> getTenantById(
        @PathVariable("id")
        @Parameter(description = "用户唯一标识，采用UUID格式，32位16进制字符串",
                required = true,
                example = "123e4567-e89b-12d3-a456-426614174000")
        UUID id) {
    // 方法实现
}

```java
@GetMapping()
@Operation(summary = "查询", description = "查询租户列表")
public Response<PageResult<Tenant>> getTenants(
        TenantQueryDTO query,
        Pageable pageable
) {
    // 方法实现
}
```

## 6. 与 ApiPost 集成使用

ApiPost 是一个功能强大的 API 调试和管理工具，支持与 Swagger/OpenAPI 集成，方便开发者导入和管理 API 文档。

### 6.1 前提条件

1. 启动项目，确保 Swagger 文档可以正常访问
2. 安装 ApiPost 客户端（可以从官网下载）

### 6.2 导入 OpenAPI 文档到 ApiPost

#### 6.2.1 手动导入

1. 打开 ApiPost 客户端
2. 点击左侧菜单栏的「导入/导出」→「导入项目（手动）」
3. 选择「OpenAPI/Swagger」格式
4. 输入 OpenAPI 文档的 URL：`http://localhost:8080/v3/api-docs`
5. 点击「导入」按钮，等待导入完成

#### 6.2.2 实时同步

ApiPost 支持实时同步 OpenAPI 文档，当 API 发生变化时，会自动更新文档。

1. 打开 ApiPost 客户端
2. 点击左侧菜单栏的「导入/导出」→「同步数据（实时）」
3. 点击「新建」按钮，创建一个同步任务
4. 填写以下信息：
   - **数据源名称**：例如 "springdoc"
   - **数据源格式**：选择 "openApi"
   - **数据源 URL**：`http://localhost:8080/v3/api-docs`
   - **导入频率**：根据需要选择，例如「手动触发」
5. 在「接口」标签页中，设置：
   - **选择目录**：选择要导入到的目录，例如「根目录」
   - **接口覆盖方式**：选择「接口相同且目录相同时覆盖数据」
   - **路径处理方式**：根据需要开启「接口路径加上 basePath」和「接口路径加上 host」
6. 点击「保存」按钮，完成同步任务的创建
7. 点击「立即导入」按钮，开始第一次同步

### 6.3 使用 ApiPost 测试 API

导入完成后，你可以在 ApiPost 中测试 API：

1. 在左侧菜单栏中找到导入的 API 接口
2. 点击接口名称，打开接口测试页面
3. 填写请求参数
4. 点击「发送」按钮，查看响应结果

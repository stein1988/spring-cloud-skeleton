# 启动问题

## 1.1 端口被占用

**问题描述**：启动应用时出现端口被占用的错误。

**解决方法**：
- 修改配置文件中的端口号：
  ```yaml
  server:
    port: 8081  # 改为其他未被占用的端口
  ```
- 或者在启动命令中指定端口：
  ```bash
  java -jar app.jar --server.port=8081
  ```
- 检查并关闭占用端口的进程。

## 1.2 应用启动失败，提示找不到主类

**问题描述**：运行 `java -jar` 命令时提示找不到主类。

**解决方法**：
- 检查 Maven 打包配置，确保指定了正确的主类：
  ```xml
  <build>
    <plugins>
      <plugin>
        <groupId>org.apache.maven.plugins</groupId>
        <artifactId>maven-jar-plugin</artifactId>
        <configuration>
          <archive>
            <manifest>
              <mainClass>com.lonbon.demo.App</mainClass>
            </manifest>
          </archive>
        </configuration>
      </plugin>
    </plugins>
  </build>
  ```

## 1.3 启动时加载不到配置文件

**问题描述**：应用启动时无法加载配置文件，导致配置项为默认值。

**解决方法**：
- 确保配置文件位于正确的位置（默认在 resources 目录下）。
- 检查配置文件名称是否正确（默认支持 app.yml、app.properties 等）。
- 检查配置文件格式是否正确（YAML 或 Properties 格式）。

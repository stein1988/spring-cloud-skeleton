# 第一课：项目框架基础与多模块架构

> **课程时长**: 45分钟（含代码演示）  
> **课程目标**: 让学员了解项目的基础技术栈、核心框架组件，以及多模块代码架构体系  
> **前置要求**: 具备Java基础，了解Spring Boot基本概念

---

## 项目设计目的

在开始学习技术细节之前，我们先了解为什么要构建这个开发体系：

### 1. 规范化开发，减少错误率

- **统一代码结构**: 所有业务模块遵循相同的分层架构（Entity → Repository → Service → Controller）
- **标准化组件**: 提供经过验证的基础组件，避免重复造轮子
- **类型安全**: 编译期检查，减少运行时错误
- **自动化处理**: 主键生成、审计字段、逻辑删除等通用功能自动处理

### 2. 基本功能封装，提高开发效率

- **开箱即用**: 基础CRUD操作只需继承基类即可使用
- **快速开发**: 新增一个完整业务模块只需6步
- **减少样板代码**: Lombok + MapStruct-Plus 大幅简化代码
- **专注业务**: 开发者可以将更多精力放在业务逻辑上，而非基础设施

### 3. 持续演进，不断封装业务功能

- **可扩展架构**: 分层设计便于后续添加新功能
- **组件沉淀**: 将通用业务功能逐步封装到module-base
- **技术升级**: 采用最新技术栈，保持架构先进性

### 1.2 ORM框架：Easy-Query（8分钟）

#### 什么是Easy-Query？

Easy-Query是一个轻量级、高性能的Java ORM框架，提供类型安全的查询API，相比传统MyBatis/JPA有以下优势：

- **类型安全**: 编译期检查，避免SQL拼写错误
- **Lambda表达式**: 链式调用，代码可读性强
- **代理实体**: 自动生成Proxy类，支持导航属性
- **高性能**: 底层优化，查询效率高

#### 项目中的使用方式

##### 1. 实体定义（Entity）

```java
@Data
@FieldNameConstants
@EqualsAndHashCode(callSuper = true)
@Table(value = "sys_user", ignoreProperties = {BaseEntity.Fields.tenantId, BaseEntity.Fields.departmentId})
@EntityProxy  // 启用代理生成
public class User extends BaseEntity implements ProxyEntityAvailable<User, UserProxy> {
    
    private String username;
    private String passwordHash;
    private UUID currentTenantId;
    
    // 导航属性（关联查询）
    @Navigate(value = RelationTypeEnum.OneToOne, 
              selfProperty = Fields.currentTenantId, 
              targetProperty = BaseEntity.Fields.id)
    private Tenant currentTenant;
}
```

##### 2. 基础实体（BaseEntity）

所有业务实体继承`BaseEntity`，包含通用字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键，使用UUIDv7算法 |
| tenantId | UUID | 租户ID（多租户支持） |
| departmentId | UUID | 部门ID |
| createTime | OffsetDateTime | 创建时间（UTC） |
| updateTime | OffsetDateTime | 更新时间（UTC） |
| createBy | UUID | 创建人ID |
| updateBy | UUID | 更新人ID |
| isDelete | Boolean | 逻辑删除标志 |
| version | Integer | 乐观锁版本号 |

##### 3. 仓库实现（Repository）

继承`EasyQueryRepository`基类：

```java
@Repository
public class UserRepositoryImpl extends EasyQueryRepository<User, UserProxy, UserProxy.UserProxyFetcher>
        implements UserRepository {

    private static final Map<String, SQLActionExpression2<IncludeContext, UserProxy>> navigateMap = new HashMap<>();

    static {
        navigateMap.put(User.Fields.currentTenant, (c, p) -> {
            c.query(p.currentTenant());
        });
    }

    public UserRepositoryImpl(EasyEntityQuery easyEntityQuery) {
        super(easyEntityQuery, User.class, proxy -> proxy.FETCHER);
    }

    @Override
    protected Map<String, SQLActionExpression2<IncludeContext, UserProxy>> getNavigateMap() {
        return navigateMap;
    }
}
```

##### 4. 查询示例

```java
// 根据ID查询
Optional<User> user = repository.getById(id);

// 条件查询
List<User> users = repository.getAll(p -> p.username().eq("admin"));

// 分页查询
PageResult<User> result = repository.getPagination(queryDto, pageable);

// 包含关联数据查询
Optional<User> user = repository.getById(id, List.of("currentTenant", "roles"), false);
```

### 1.3 对象转换框架：MapStruct-Plus（5分钟）

#### 什么是MapStruct-Plus？

MapStruct-Plus是MapStruct的增强版本，提供：

- **编译期代码生成**: 运行时零开销
- **Lombok兼容**: 与Lombok无缝集成
- **增强注解**: 简化配置
- **类型安全**: 编译期检查

#### 项目中的使用方式

##### 1. Maven配置

在父POM中配置注解处理器：

```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-compiler-plugin</artifactId>
    <configuration>
        <annotationProcessorPaths>
            <!-- Lombok -->
            <path>
                <groupId>org.projectlombok</groupId>
                <artifactId>lombok</artifactId>
            </path>
            <!-- MapStruct-Plus -->
            <path>
                <groupId>io.github.linpeilie</groupId>
                <artifactId>mapstruct-plus-processor</artifactId>
            </path>
            <!-- Lombok-MapStruct绑定 -->
            <path>
                <groupId>org.projectlombok</groupId>
                <artifactId>lombok-mapstruct-binding</artifactId>
            </path>
        </annotationProcessorPaths>
    </configuration>
</plugin>
```

##### 2. 在Service中使用

```java
@Transactional(rollbackFor = Exception.class)
public abstract class EntityServiceImpl<T, TProxy> implements EntityService<T, TProxy> {
    
    protected final Converter converter;  // MapStruct-Plus转换器
    protected final Repository<T, TProxy> repository;
    
    // 创建实体：DTO -> Entity
    public T createEntity(Object createDto) {
        T entity = converter.convert(createDto, entityType);
        repository.insert(entity);
        return entity;
    }
    
    // 更新实体：DTO -> Entity
    public void updateEntity(UUID id, Object updateDto) {
        this.updateEntity(id, (Function<T, T>) entity -> converter.convert(updateDto, entity));
    }
}
```

##### 3. 优势对比

| 方案 | 性能 | 类型安全 | 代码量 | 维护成本 |
|------|------|---------|--------|---------|
| 手动转换 | 高 | 高 | 多 | 高 |
| BeanUtils | 低 | 低 | 少 | 中 |
| MapStruct | 高 | 高 | 少 | 低 |
| MapStruct-Plus | 高 | 高 | 极少 | 极低 |

## 第二部分：多模块代码架构（约20分钟，含代码演示）

### 2.1 多模块架构设计（5分钟）

#### 模块划分原则

```
spring-cloud-skeleton/
├── module-base/                    # 基础模块
│   ├── entity/                     # 基础实体
│   ├── repository/                 # 基础仓库接口
│   ├── service/                    # 基础服务
│   ├── dto/                        # 通用DTO
│   ├── exception/                  # 异常处理
│   └── response/                   # 统一响应
│
├── module-user/                    # 用户模块（业务模块示例）
│   ├── domain/                     # 领域层
│   │   ├── entity/                 # 业务实体
│   │   ├── repository/             # 仓库接口
│   │   ├── service/                # 服务接口
│   │   ├── dto/                    # 数据传输对象
│   │   └── value_object/           # 值对象
│   ├── infrastructure/             # 基础设施层
│   │   ├── repository/             # 仓库实现
│   │   └── config/                 # 配置类
│   ├── application/                # 应用层
│   │   └── service/                # 服务实现
│   └── api/                        # 接口层
│       └── controller/             # 控制器
```

#### 模块职责

| 模块 | 职责 | 依赖关系 |
|------|------|---------|
| module-base | 提供通用组件、工具类、基础抽象 | 无业务依赖 |
| module-user | 具体业务实现 | 依赖module-base |

### 2.2 模块内代码架构体系（10分钟）

采用**分层架构**（Layered Architecture），遵循**依赖倒置原则**（DIP）：

```
┌─────────────────────────────────────────┐
│           API层（Controller）            │  ← 对外接口
├─────────────────────────────────────────┤
│        Application层（ServiceImpl）      │  ← 业务编排
├─────────────────────────────────────────┤
│          Domain层（Entity/Service）      │  ← 核心业务逻辑
├─────────────────────────────────────────┤
│     Infrastructure层（Repository）       │  ← 数据访问
└─────────────────────────────────────────┘
```

#### 各层职责

##### 1. Domain层（领域层）

**核心职责**: 定义业务实体、服务接口、DTO

```java
// 实体定义
@EntityProxy
public class User extends BaseEntity implements ProxyEntityAvailable<User, UserProxy> {
    private String username;
    private String passwordHash;
    // ...
}

// 服务接口
public interface UserService {
    User createEntity(Object createDto);
    void updateEntity(UUID id, Object updateDto);
    void deleteEntity(UUID id);
    Optional<User> getEntityById(UUID id);
}

// 仓库接口
public interface UserRepository extends Repository<User, UserProxy> {
    // 可扩展自定义查询方法
}
```

##### 2. Infrastructure层（基础设施层）

**核心职责**: 实现仓库接口，处理数据访问

```java
@Repository
public class UserRepositoryImpl extends EasyQueryRepository<User, UserProxy, UserProxy.UserProxyFetcher>
        implements UserRepository {

    public UserRepositoryImpl(EasyEntityQuery easyEntityQuery) {
        super(easyEntityQuery, User.class, proxy -> proxy.FETCHER);
    }
}
```

##### 3. Application层（应用层）

**核心职责**: 实现服务接口，编排业务流程

```java
@Service
public class UserServiceImpl extends EntityServiceImpl<User, UserProxy> 
        implements UserService {

    public UserServiceImpl(Converter converter, UserRepository repository) {
        super(converter, repository, User.class);
    }
}
```

##### 4. API层（接口层）

**核心职责**: 提供RESTful API，处理HTTP请求

```java
@RestController
@RequestMapping("/users")
@Tag(name = "用户", description = "用户操作")
public class UserController {

    @Resource
    private UserService userService;

    @PostMapping
    @Operation(summary = "创建", description = "创建用户")
    public Response<UUID> create(@RequestBody @Validated @NotNull UserCreateDTO user) {
        User createdUser = userService.createEntity(user);
        return Response.success(createdUser.getId(), "User created successfully");
    }

    @GetMapping("/{id}")
    @Operation(summary = "获取", description = "获取用户")
    public Response<User> getUserById(@PathVariable("id") UUID id) {
        Optional<User> user = userService.getEntityById(id);
        return user.map(Response::success).orElseGet(() -> Response.error("User not found"));
    }
}
```

## 课程总结（5分钟）

### 核心要点回顾

1. **技术栈选型**: Spring Boot 4 + Java 25 + Easy-Query + MapStruct-Plus
2. **ORM框架**: Easy-Query提供类型安全、高性能的数据访问
3. **对象转换**: MapStruct-Plus实现编译期、零开销的对象映射
4. **多模块架构**: module-base（基础）+ module-xxx（业务）
5. **分层架构**: API → Application → Domain → Infrastructure

### 架构优势

- ✅ **高内聚低耦合**: 各层职责清晰，依赖关系明确
- ✅ **可扩展性强**: 新增业务实体只需6步
- ✅ **类型安全**: 编译期检查，减少运行时错误
- ✅ **高性能**: MapStruct编译期生成，Easy-Query优化查询

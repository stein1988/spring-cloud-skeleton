ORM框架：Easy-Query

轻量级、高性能的Java ORM框架

✅
类型安全：编译期检查，避免SQL拼写错误
✅
Lambda表达式：链式调用，代码可读性强
✅
代理实体：自动生成Proxy类，支持导航属性
✅
高性能：底层优化，查询效率高

ORM框架对比

MyBatis / JPA

  • XML配置或字符串SQL
  • 字段名硬编码
  • 关联查询配置复杂
  • 学习成本较低
  • 业界常用组件

Easy-Query

  • 类型安全Lambda查询
  • 编译期检查错误
  • 自动生成Proxy类
  • 导航属性简化关联
  • 学习成本较高可通过封装降低

分层架构设计

遵循依赖倒置原则（DIP）

API层（Controller）
对外接口 · RESTful API · HTTP请求处理
⬇
Application层（ServiceImpl）
业务编排 · 流程控制 · 事务管理
⬇
Domain层（Entity / Service / Repository）
核心业务逻辑 · 实体定义 · 接口抽象
⬇
Infrastructure层（RepositoryImpl）
数据访问 · 仓库实现 · 外部集成

Domain层 — 核心业务逻辑

📦
业务实体（Entity）：继承BaseEntity，定义业务字段和导航属性
📋
服务接口（Service）：定义业务方法契约，如CRUD操作
🗄️
仓库接口（Repository）：定义数据访问抽象，继承基础Repository
📝
DTO和值对象：CreateDTO、UpdateDTO、QueryDTO等

关键原则：Domain层只定义接口，不包含实现细节，确保业务逻辑的纯粹性

Infrastructure层 — 数据访问

🔧
实现仓库接口：继承 EasyQueryRepository，提供数据访问实现
🔗
导航属性映射：配置 NavigateMap，定义关联查询关系
⚙️
配置类：模块特定的基础设施配置
Repository 接口
Domain层定义
→ 实现
RepositoryImpl
Infrastructure层实现
→ 继承
EasyQueryRepository
基础仓库

Application层 — 业务编排

🎯
实现服务接口：继承 EntityServiceImpl，获得基础CRUD能力
🔄
编排业务流程：组合多个Repository操作，实现复杂业务
💳
事务管理：@Transactional 保证数据一致性
🔀
对象转换：使用 Converter 完成 DTO ↔ Entity 转换

核心优势：继承 EntityServiceImpl 后，基础CRUD无需编写任何代码

API层 — RESTful接口

🌐
RESTful API：定义HTTP接口，处理请求和响应
📝
参数校验：@Validated 注解驱动参数验证
📖
API文档：@Tag、@Operation 注解自动生成Swagger文档
📦
统一响应：Response 封装统一返回格式

步骤 1-3：实体和仓库

Step 1 · 创建实体

继承 BaseEntity → 定义字段 → @EntityProxy → @Table

Step 2 · 仓库接口

继承 Repository<Department, DepartmentProxy>

Step 3 · 仓库实现

继承 EasyQueryRepository → 配置 Fetcher → 注册导航属性映射

继承关系：EasyQueryRepository → Repository接口 → 业务Repository

步骤 4-6：服务和控制器

Step 4 · 服务接口

定义业务方法：createEntity、getEntityById 等

Step 5 · 服务实现

继承 EntityServiceImpl → 注入 Converter + Repository → 自动获得CRUD

Step 6 · 控制器

@RestController → 注入Service → 定义端点 → @Tag配置文档

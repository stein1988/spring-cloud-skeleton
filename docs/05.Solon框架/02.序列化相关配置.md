# 序列化相关配置

本文档介绍在 Solon 框架下，业务系统的序列化相关配置，包括 UUID 类型转换器和 JSON 序列化配置。

## 1. UUID 类型转换器配置

在 Web 开发时，有些自定义类型框架没法自动处理，需要自定义转换器处理。UUID 是常用的主键类型，Solon 框架默认不支持将路径参数或表单参数中的字符串自动转换为 UUID 类型。

### 1.1 UUID 转换器实现

参考：[Solon 定制 Converter 处理特别类型转换](https://solon.noear.org/article/567)

```java
package com.lonbon.cloud.base.config;

import org.noear.solon.core.convert.Converter;
import org.noear.solon.core.exception.ConvertException;

import java.util.UUID;

/**
 * UUID 转换器，用于接收请求时，自动将字符串转换为 UUID 类型
 * 注意：
 * 1. 这里不能使用 @Component 注解自动注入，否则easy-query框架自动生成的实体类会缺少id字段的相关函数，原因未知，
 * 因此，需要通过插件或手动注册到 Solon 容器中
 * 参考：https://solon.noear.org/article/567  定制 Converter 处理特别类型转换
 */
//@Component
public class UUIDConverter implements Converter<String, UUID> {

    @Override
    public UUID convert(String value) throws ConvertException {
        try {
            return UUID.fromString(value);
        }  catch (Exception ex) {
            throw new ConvertException(ex);
        }
    }
}
```

### 1.2 UUID 转换器注册

#### 1.2.1 通过插件自动注册

在基础模块的 Solon 插件中自动注册：

```java
package com.lonbon.cloud.base.plugin;

import com.lonbon.cloud.base.config.UUIDConverter;
// 其他导入...

public class BaseSolonPlugin implements Plugin {
    @Override
    public void start(AppContext context) throws Throwable {
        // 注册 UUID 类型转换器
        context.beanMake(UUIDConverter.class);
        // 其他组件注册...
    }
}
```

#### 1.2.2 手动注册

在应用入口类中手动注册转换器：

```java
package com.lonbon.cloud.user.app;

import com.lonbon.cloud.base.config.UUIDConverter;
import org.noear.solon.Solon;
import org.noear.solon.annotation.Import;
import org.noear.solon.annotation.SolonMain;

@SolonMain
@Import(scanPackages = {"com.lonbon.cloud.user", "com.lonbon.cloud.base"})
public class App {
    public static void main(String[] args) {
        Solon.start(App.class, args, app->{
            app.converters().register(new UUIDConverter());
        });
    }
}
```

### 1.3 控制器中使用 UUID 参数

配置完成后，控制器方法可以直接接收 UUID 类型的路径参数：

```java
@Get
@Mapping("/{id}")
public Response<Tenant> getTenantById(@Path("id") UUID id) {
    // ...
}
```

## 2. JSON 序列化配置

Solon 框架默认使用 snack4 作为 JSON 序列化器，可以进行定制化配置。

### 2.1 OffsetDateTime 序列化配置

参考：[solon-serialization-snack4](https://solon.noear.org/article/1200)

在业务系统中，通常使用带时区的时间戳 `OffsetDateTime`，需要配置其序列化格式为 ISO 8601 格式：

```java
package com.lonbon.cloud.base.config;

import org.noear.solon.annotation.Bean;
import org.noear.solon.annotation.Component;
import org.noear.solon.serialization.snack4.Snack4StringSerializer;

import java.time.OffsetDateTime;
import java.time.format.DateTimeFormatter;

/**
 * 序列化配置类，用于定制序列化器的行为，用于控制接口返回的 JSON 字段格式
 * 1. 定制 OffsetDateTime 类型的序列化格式为 ISO 8601 格式
 * 参考：https://solon.noear.org/article/1200 solon-serialization-snack4 高级格式化定制
 */
@Component
public class SerializerConfiguration {

    @Bean
    public void config(Snack4StringSerializer serializer) {
        serializer.addEncoder(OffsetDateTime.class, s -> s.format(DateTimeFormatter.ISO_OFFSET_DATE_TIME));
    }
}
```

### 2.2 序列化配置的自动注册

序列化配置通过 BaseSolonPlugin 自动注册到应用中：

```java
package com.lonbon.cloud.base.plugin;

import com.lonbon.cloud.base.config.SerializerConfiguration;
// 其他导入...

public class BaseSolonPlugin implements Plugin {
    @Override
    public void start(AppContext context) throws Throwable {
        // 注册序列化配置组件
        context.beanMake(SerializerConfiguration.class);
        // 其他组件注册...
    }
}
```

### 2.3 常用序列化配置项

在 `app.yml` 中也可以配置序列化相关选项：

```yaml
solon.serialization.json:
  dateAsFormat: 'yyyy-MM-dd HH:mm:ss' # 配置日期格式（默认输出为时间戳，对 Date、LocalDateTime 有效）
  dateAsTimeZone: 'GMT+8' # 配置时区
  dateAsTicks: false # 将 date 转为毫秒数（和 dateAsFormat 二选一）
  longAsString: true # 将 long 型转为字符串输出（默认为 false）
  boolAsInt: false   # 将 bool 型转为字符串输出（默认为 false）
  enumAsName: false # 枚举使用名字
```

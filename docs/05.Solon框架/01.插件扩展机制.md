# Solon 插件扩展机制

本文档介绍 Solon 框架的插件扩展机制，以及如何利用该机制实现基础模块中公共组件的自动依赖注入。

## 1. 插件扩展机制简介

Solon 的插件扩展机制是基于“插件”+“配置声明”实现的解耦扩展机制，类似于 Spring Factories 和 Java SPI。它的核心作用是为模块获得应用启动入口，并参与应用生命周期。

在业务系统中，我们可以利用插件机制将基础模块（module-base）中的公共组件（如序列化配置、类型转换器、实体拦截器等）自动注册到应用中，无需在每个业务模块的启动类中手动配置。

## 2. 插件实现

### 2.1 插件实现类

```java
package com.lonbon.cloud.base.plugin;

import com.lonbon.cloud.base.config.EasyQueryConfiguration;
import com.lonbon.cloud.base.config.SerializerConfiguration;
import com.lonbon.cloud.base.config.UUIDConverter;
import com.lonbon.cloud.base.entity.DefaultEntityInterceptor;
import com.lonbon.cloud.base.entity.DefaultLogicDeleteStrategy;
import com.lonbon.cloud.base.entity.OffsetDateTimeTypeHandler;
import com.lonbon.cloud.base.entity.UUIDPrimaryKeyGenerator;
import com.lonbon.cloud.base.exception.ExceptionFilter;
import org.noear.solon.core.AppContext;
import org.noear.solon.core.Plugin;

public class BaseSolonPlugin implements Plugin {
    @Override
    public void start(AppContext context) throws Throwable {
        context.beanMake(SerializerConfiguration.class);
        context.beanMake(ExceptionFilter.class);
        context.beanMake(UUIDConverter.class);

        context.beanMake(DefaultEntityInterceptor.class);
        context.beanMake(DefaultLogicDeleteStrategy.class);
        context.beanMake(OffsetDateTimeTypeHandler.class);
        context.beanMake(UUIDPrimaryKeyGenerator.class);
        context.beanMake(EasyQueryConfiguration.class);
    }
}
```

### 2.2 插件配置文件

在 `src/main/resources/META-INF/solon/` 目录下创建插件配置文件：

文件名：`com.lonbon.cloud.base.plugin.BaseSolonPlugin.properties`

```properties
solon.plugin=com.lonbon.cloud.base.plugin.BaseSolonPlugin
solon.plugin.priority=0
```

配置说明：
- `solon.plugin`：指定插件实现类的全限定名
- `solon.plugin.priority`：插件优先级，数值越大越优先，默认为 0

## 3. 当前组件说明

BaseSolonPlugin 插件中注册了以下公共组件：

| 组件 | 作用 |
|------|------|
| SerializerConfiguration | 定制 JSON 序列化行为，如 OffsetDateTime 的格式配置 |
| ExceptionFilter | 全局异常过滤器，处理请求过程中的异常 |
| UUIDConverter | UUID 类型转换器，将请求参数中的字符串转换为 UUID |
| DefaultEntityInterceptor | 实体拦截器，自动设置创建时间、更新时间等字段 |
| DefaultLogicDeleteStrategy | 逻辑删除策略，实现数据的软删除 |
| OffsetDateTimeTypeHandler | OffsetDateTime 类型处理器，处理 PostgreSQL timestamptz 类型 |
| UUIDPrimaryKeyGenerator | UUID 主键生成器，使用 UUIDv7 算法生成主键 |
| EasyQueryConfiguration | Easy-Query 框架配置 |

## 4. 插件的排除

如果引入某个插件包后需要排除某个插件，可以使用以下方式：

### 配置方式

```yaml
solon.plugin.exclude:
  - "{PluginImpl}"
```

### 编码方式

```java
public class App {
    public static void main(String[] args){
        Solon.start(App.class, args, app -> {
            app.pluginExclude(BaseSolonPlugin.class);
        });
    }
}
```
